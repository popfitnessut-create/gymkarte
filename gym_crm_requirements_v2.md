# 会員カルテ管理アプリ 要件定義書 v2
> Claude Code向け実装指示書

---

## プロジェクト概要

**アプリ名**: GymKarte  
**目的**: パーソナルジムの会員情報・セッション・回数券・日次カルテ・トレーニング記録を一元管理する社内専用デスクトップアプリ  
**運用環境**: PC（Windows / Mac）  
**技術スタック**: Electron + React + SQLite（ローカルDB）  
**オフライン動作**: 必須（インターネット不要）  
**デスクトップアイコン**: 必須（electron-builderでインストーラー生成）

---

## 技術スタック詳細

```
フレームワーク : Electron（デスクトップアプリ化）
UI           : React + Tailwind CSS
DB           : SQLite（better-sqlite3）※ローカル保存
ビルド        : electron-builder（.exe / .dmgインストーラー生成）
状態管理      : Zustand
グラフ        : Recharts
検索          : Fuse.js（高精度あいまい検索）
アイコン      : Lucide React
```

## 画面構成

### 1. ダッシュボード（TOP画面）

- 本日の来店予定・来店済み人数
- 回数券残数アラート（残3回以下の会員をハイライト表示）
- 直近7日間の来店数グラフ（棒グラフ）
- 全会員数・アクティブ会員数のサマリーカード

---

### 2. 会員一覧画面

- 会員の一覧テーブル表示
- **高精度検索バー**（後述）
- フィルター（アクティブ / 休会 / 退会）
- 新規会員登録ボタン
- 各会員行をクリック → 会員詳細（カルテ）画面へ遷移
- **「マルチ展開」ボタン**：チェックした会員（最大10名）を同時展開モードで開く（後述）

---

### 3. 会員詳細（カルテ）画面

#### 3-1. 基本情報タブ
| 項目 | 内容 |
|------|------|
| 氏名（漢字・フリガナ） | テキスト入力 |
| 生年月日・年齢（自動計算） | 日付入力 |
| 性別 | セレクト |
| 電話番号・メールアドレス | テキスト入力 |
| 入会日 | 日付入力 |
| ステータス | アクティブ / 休会 / 退会 |
| 目標（自由記述） | テキストエリア |
| 健康状態・既往歴 | テキストエリア |
| 特記事項・備考 | テキストエリア |

#### 3-2. 回数券タブ
- 回数券の購入履歴一覧（購入日・枚数・有効期限・購入金額）
- 現在の残回数（大きく表示）
- 新規購入ボタン（枚数・有効期限・金額を入力）
- 使用履歴（セッション記録と連動して自動減算）
- 残3回以下で警告表示

#### 3-3. セッション記録タブ
- セッション一覧（日付の降順）
- 各セッションに記録する項目：
  - 来店日時
  - セッション人数（1〜4名：セミパーソナル対応）
  - 担当トレーナー名
  - 回数券消費（1回自動カウント、手動調整可）
  - 鍛えた部位（ボディマップで選択）
  - トレーニングメニュー（種目名・重量・セット数・レップ数）
  - フォームメモ・コーチコメント（自由記述）
  - 次回へのメモ（自由記述）
- 新規セッション記録ボタン

#### 3-4. 日次カルテタブ ※新規追加
セッションがない日も含め、日単位で体調・生活習慣・コーチメモを記録できるログ機能。

| 項目 | 入力形式 |
|------|---------|
| 記録日 | 日付（デフォルト：今日） |
| 体重(kg) | 数値入力（任意） |
| 体脂肪率(%) | 数値入力（任意） |
| 体調 | 5段階評価（😞〜😊 または 1〜5） |
| 睡眠時間(h) | 数値入力（任意） |
| 睡眠の質 | 5段階評価 |
| 食事メモ | テキストエリア（朝・昼・夜・間食） |
| 水分摂取量(ml) | 数値入力（任意） |
| 本人コメント | テキストエリア（気になること・自己申告） |
| トレーナー所見 | テキストエリア（次回への引き継ぎとは別管理） |

- カレンダービューで日次ログの有無を一覧表示（記録済みの日にドット表示）
- 体重・体脂肪率の推移グラフを分析タブと連携

#### 3-5. 分析タブ
- 来店頻度グラフ（月別・折れ線グラフ）
- 鍛えた部位の割合グラフ（ドーナツグラフ）
- 重量の伸び推移（種目別・折れ線グラフ）
- 体重・体脂肪率の推移グラフ（日次カルテデータから）
- 総来店回数・平均来店ペース

---

### 4. セッション記録入力モーダル

#### ボディマップ（部位選択UI）
- 人体の前面・背面の簡易シルエット図をSVGで表示
- 各部位をクリックで選択（複数選択可）
- 選択された部位がハイライト表示される
- 対応部位：
  - 前面：胸・腹・前腕・大腿四頭筋・肩（前）
  - 背面：背中（広背筋）・僧帽筋・ハムストリングス・臀部・ふくらはぎ・肩（後）・上腕二頭筋・上腕三頭筋

#### トレーニングメニュー入力
- 種目を行単位で追加（＋ボタンで行追加）
- 各行：種目名（テキスト or プリセット選択）・重量(kg)・セット数・レップ数
- 種目プリセット例：ベンチプレス、スクワット、デッドリフト、ラットプルダウン、etc.

### 5. 高精度会員名検索機能 ※新規追加

**要件**
- 漢字・ひらがな・カタカナ・ローマ字すべてで検索可能
- あいまい検索対応（例：「たなか」→「田中」「谷中」なども候補表示）
- 入力途中でリアルタイムに候補が絞り込まれるインクリメンタルサーチ
- 変換ミス・誤字に対して類似候補を提示（Fuse.jsのfuzzy search）
- 検索対象フィールド：氏名（漢字）・フリガナ・電話番号・会員ID
- 検索結果にはサムネイル情報（残回数・最終来店日）も表示

**実装方針**
```
ライブラリ : Fuse.js
検索対象   : members テーブルの name, furigana, phone, id
オプション  : threshold: 0.4（あいまい度調整）, includeScore: true
表示       : スコア順に最大10件表示、キーワードハイライト付き
```

### 6. マルチカルテ同時展開機能 ※新規追加

**概念**
1枠に最大4名が同時参加するセミパーソナル運営に対応し、同じ時間帯に来店した会員のカルテを並べて確認・記録できる機能。

**仕様**
- 会員一覧でチェックボックスにより最大10名を選択
- 「マルチ展開」ボタンで専用レイアウトに切り替え
- 展開レイアウト：
  - 2〜4名：横並びカード表示（各カードに氏名・残回数・前回メモ・今回入力欄）
  - 5〜10名：グリッド表示（2列×最大5行）
- 各カードで独立して以下を操作可能：
  - 簡易セッション記録（鍛えた部位・コメント・回数券消費）
  - 日次カルテ簡易入力（体調・本人コメント・トレーナー所見）
- 「一括保存」ボタンで全員分を同時保存
- 同一枠の会員に同じトレーニングメニューを「一括コピー」するボタンも設置
- 各カードからフルカルテ（詳細画面）への遷移リンクあり

**UI補足**
- カード上部に氏名・残回数・前回来店日を常時表示
- 残回数が3以下のカードは赤枠でアラート表示
- カードの並び順はドラッグ&ドロップで入れ替え可能

### 7. Excelインポート機能 ※新規追加

**目的**：既存の160名分カルテデータをExcelから一括移行する

**対応フォーマット**：.xlsx / .csv

**インポートフロー**
1. 設定画面から「データインポート」を選択
2. Excelファイルをドラッグ&ドロップまたはファイル選択
3. Excelの列と本アプリのフィールドをマッピング（対応表をGUI上で設定）
4. プレビュー表示（先頭10件を確認）
5. エラー検出（必須項目の欠落・日付フォーマット不正など）
6. 「インポート実行」ボタンで一括取込
7. インポートログを表示（成功件数・スキップ件数・エラー内容）

**対応する移行データ項目**
- 基本情報（氏名・フリガナ・生年月日・性別・連絡先・入会日・ステータス）
- 目標・健康状態・備考
- 回数券情報（残回数・有効期限）
- ※セッション履歴は別シートから任意でインポート可能

**実装方針**
```
ライブラリ : SheetJS（xlsx）
処理       : Electronのmainプロセスで読み込み→DBに一括INSERT
エラー処理 : バリデーション後、エラー行のみスキップしてログ出力
```

## データベース設計

```sql
-- 会員テーブル
members (
  id, name, furigana, birthdate, gender,
  phone, email, joined_at, status,
  goal, health_notes, notes, created_at, updated_at
)

-- 回数券テーブル
tickets (
  id, member_id, purchased_at, total_count,
  remaining_count, expires_at, price, notes
)

-- セッションテーブル
sessions (
  id, member_id, ticket_id, session_date,
  participant_count, trainer_name,
  coach_comment, next_memo, created_at
)

-- 鍛えた部位テーブル
session_muscles (
  id, session_id, muscle_name
)

-- トレーニングメニューテーブル
session_exercises (
  id, session_id, exercise_name,
  weight_kg, sets, reps, order_index
)

-- 日次カルテテーブル ※新規
daily_logs (
  id, member_id, log_date,
  weight_kg, body_fat_pct,
  condition_score,        -- 1〜5
  sleep_hours, sleep_quality_score,
  meal_notes,             -- 朝・昼・夜・間食（JSON or テキスト）
  water_ml,
  member_comment,         -- 本人コメント
  trainer_note,           -- トレーナー所見
  created_at, updated_at
)

-- トレーナーマスタ
trainers (id, name, active)

-- 種目マスタ
exercise_presets (id, name, category)
```

## UI/UXデザイン方針

- カラー：ダークネイビー×ホワイト×アクセントカラー（電気ブルーまたはゴールド）
- フォント：日本語対応（Noto Sans JP）
- レイアウト：左サイドバー（ナビゲーション）＋メインコンテンツエリア
- 操作性：クリック数を最小化。カルテ開いて即セッション記録できる動線
- マルチ展開時は全画面モードで横広レイアウトに自動切り替え
- レスポンシブ：PC固定（1280px以上）
