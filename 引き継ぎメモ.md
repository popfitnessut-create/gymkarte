## 2. ⚠️ 一番大事な落とし穴：libSQL の名前付きパラメータが壊れている

better-sqlite3 を外して **libsql のみ**にした際に発覚した重大バグ。

- `@name` プレースホルダ＋オブジェクト渡し（`stmt.run({ name: ... })`）は
  **黙って NULL を書き込む**（エラーも出ない）。`@キー付き`でもダメ。
- **positional `?` だけが正しく動く**（test-bind.js で実証済み）。
- これが原因で「備考を編集して保存すると消える」バグが起きていた。

### 対処（実装済み）
`electron/db.js` に `applyNamedParamShim(database)` を追加。
`prepare()` をラップして `@name` → `?` に変換し、オブジェクト引数を
名前の順に positional 配列へ並べ替えてから渡す。同期モード・ローカルモード
両方の `new Database(...)` 直後に適用済み。

➡️ **今後 db.js / ipc.js で新しいSQLを書くときの鉄則**:
このshimのおかげで `@name` も動くようになっているが、shimを外したり
別の箇所でprepareを直接呼ぶ場合は positional `?` を使うこと。

## 3. 自動アップデート（electron-updater）

- 実装は `electron/updater.js` に完成済み（手動チェック＋起動4秒後の自動チェック、
  ダウンロード確認ダイアログ、ダウンロード完了→再起動ダイアログ）。
- 開発時（`app.isPackaged === false`）は無効。本番のみ動作。
- 配布は **GitHub Releases**。`package.json` の `release:win` スクリプト:
  `vite build && electron-builder --win --publish always`

### リリース手順（店舗PC = Windowsで実行）
```powershell
cd $HOME\gymkarte
git pull
$env:GH_TOKEN="ghp_（public_repo権限のclassicトークン）"
npm run release:win
```
完了後 https://github.com/popfitnessut-create/gymkarte/releases に
`vX.Y.Z` と `GymKarte Setup X.Y.Z.exe` / `latest.yml` が並べばOK。

### ⚠️ ハマったポイント（2026-06-20に解決済み）
- electron-builder は既定で GitHub リリースを **下書き(Draft)** で作る。
  下書きは公開ページに出ず、自動アップデートも拾えない。
- 対処として `package.json` の publish に **`"releaseType": "release"`** を追加済み。
  → 次回以降は最初から公開リリースになる。
- もし将来また「Releasesに出ない／There aren't any releases here」になったら、
  GitHubに**オーナーでログイン**してReleasesページの下書きを開き、
  **「Publish release」**を手動で押せば即解決する。

### バージョンを上げてリリースする流れ
1. `package.json` の `version` を上げる（例 0.3.1 → 0.3.2）
2. Macでコミット & `git push`
3. 店舗PCで上記のリリース手順を実行
4. 旧バージョンが起動中のPCで、数十秒後に自動アップデート通知が出れば成功

## 4. クラウド同期について（確認済みの仕様）

- 同期は **クラウド経由**。自宅と店舗で**回線が違っても、ネットに繋がっていれば同期される**
  （同じLAN/回線である必要はない）。
- シャットダウンしても同期設定は維持される（`sync-config.json` に保存されているため）。
- 書き込み後、ローカルには即反映。クラウドへは `db.sync()`（自動デバウンス）で反映。

---

## 5. 検証方法（サンドボックス内）

- `npm run build` はこのサンドボックスでは通らない（Mac向けnode_modules / rollupのため）。
- 代わりに構文チェック:
  - Electronの `.js`: `node --check electron/xxx.js`
  - JSX: `@babel/parser` で parse（`sourceType:'module'`, `plugins:['jsx']`）
- 実際の動作確認はユーザーがアプリを再起動して行う。
- サンドボックスからは **GitHubへのpush・Tursoへの接続はできない**（認証情報・ネット制限）。
  push やリリースはユーザーが実機（Mac/店舗PC）で実行する。

## 7. v0.3.2 の変更内容（2026-06-20・コード実装済み／リリース未）

6点の改善を実装。**サンドボックスでは構文検証のみ完了**。push・リリースは実機で要実行。

1. **動作のカクつき軽減**：recharts の全グラフに `isAnimationActive={false}` を付与
   （Dashboard 棒グラフ／AnalyticsTab 月別・部位円・重量・体重体脂肪／EvaluationTab）。
   描画時の再アニメーションを止めて再レンダリングを軽くした。
2. **お渡しアラートが消えない不具合 → 修正**：ダッシュボードのリマインダから会員を開くと、
   パフォーマンス記録表タブを**リマインダ対象月（`targetYM`）で直接開く**ようにした
   （`useStore.openMemberAt` 追加 → `MemberDetail` が初期タブ/初期月を受け取り → `EvaluationTab` に `initialYm`）。
   これでお渡し記録がリマインダと同じ月に保存され、記録後アラートが消える。
3. **HIIT子種目**：プリセットで「HIIT」を選ぶと、子種目（種目選択）／子種目重量／秒数
   （5秒刻み・最大60秒）の入力欄が出る。`session_exercises` に `seconds` `child_name` 列を追加
   （`db.js migrate()`）。`ipc.js` の insert/groupExercises/cards を秒数・子種目対応に更新。
4. **回数/秒数の選択**：通常種目は行ごとに「回数／秒数」トグルを追加。秒数は5秒刻み60秒まで。
5. **アイコンがデフォルトに戻る不具合 → 修正**：`assets/icon.ico` をマルチ解像度
   （16〜256px）で再生成。`package.json` の `win.icon`＋NSIS各アイコン、`main.js` の
   BrowserWindow `icon`、`files` への同梱を設定。
6. **アップデート通知**：`updater.js` のダイアログと `Settings.jsx` のステータス表示に
   「最新版へのバージョンアップを実行してください。」を表示。
