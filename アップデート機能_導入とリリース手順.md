# GymKarte 自動アップデート機能 — 導入とリリース手順

他端末（店舗の Windows PC）でも、Mac で作った新バージョンを自動で受け取れるようにしました。
仕組みは **GitHub Releases + electron-updater**。Mac でビルド＆アップロード → Windows が起動時に自動チェック → 通知 → 本人の確認で更新、という流れです。

---

## 何が変わったか（コード変更点）

- `package.json`
  - `electron-updater` を依存に追加
  - `build.publish` に GitHub の配信元を追加（`owner` は要設定 ↓）
  - `release:win` スクリプトを追加（ビルド＋GitHubへ公開を一括）
- `electron/updater.js`（新規）… アップデート本体のロジック
- `electron/main.js` … 起動時に `initUpdater()` を呼ぶ
- `electron/preload.js` … `window.api.updater` を公開
- `src/pages/Settings.jsx` … 設定画面に「アプリのアップデート」欄（現在バージョン表示＋「アップデートを確認」ボタン）

UX は「**通知して選択**」型です。
新バージョン検出 → 「ダウンロードしますか？」 → ダウンロード完了 → 「今すぐ再起動して更新しますか？」。
営業中に勝手に再起動しないので安心です（起動時にも自動チェックします）。

---

## 最初の一回だけ必要な準備（Mac 側）

### 1. 依存をインストール
```bash
cd ~/Desktop/gymkarte
npm install
```

### 2. GitHub リポジトリを用意
GitHub アカウントが無ければ https://github.com で作成（無料）。
その後、`gymkarte` という名前でリポジトリを新規作成します。

- **公開（Public）リポジトリが最も簡単**です（追加設定なしで Windows が更新を取得できます）。
  ソースコードが公開されますが、会員データはアプリ内（各PCのローカル）に保存されるため**顧客情報は公開されません**。
- コードを見せたくない場合は Private も可能ですが、アプリにアクセストークンを埋め込む追加設定が必要になります（必要なら別途案内します）。

ターミナルから一括で作る場合（GitHub CLI `gh` を使用、未導入なら `brew install gh`）：
```bash
cd ~/Desktop/gymkarte
git init
git add -A
git commit -m "GymKarte with auto-update"
gh auth login            # 画面の案内に従ってログイン
gh repo create gymkarte --public --source=. --push
```

### 3. `package.json` の owner を自分のユーザー名に変更
`build.publish` の `"owner": "YOUR_GITHUB_USERNAME"` を、あなたの GitHub ユーザー名に書き換えます。
（リポジトリ名を `gymkarte` 以外にした場合は `"repo"` も合わせて変更）

### 4. 公開用トークンを用意（Mac からアップロードする鍵）
GitHub の **Settings → Developer settings → Personal access tokens → Tokens (classic)** で
`repo` 権限のトークンを発行し、控えておきます。
公開（リリース）するときに環境変数 `GH_TOKEN` として渡します。

## 毎回のリリース手順（新バージョンを配る）

1. **バージョンを上げる**（ここが更新検出の鍵。必ず上げる）
   `package.json` の `"version"` を上げます。例：`0.1.0` → `0.1.1`
   ```bash
   npm version patch   # 0.1.0 -> 0.1.1（自動で書き換え）
   ```

2. **ビルドして GitHub に公開**
   ```bash
   GH_TOKEN=ここにトークン npm run release:win
   ```
   `release/` に Windows インストーラ（.exe）と `latest.yml` が作られ、GitHub の Releases に自動アップロードされます。

3. **GitHub の Releases ページでドラフトを「Publish」**
   electron-builder は下書き（draft）として上げるので、ブラウザでリポジトリの Releases を開き、該当バージョンを **Publish release** します。

これで完了。店舗の Windows PC は次回起動時（または設定→アップデートを確認）に新バージョンを検知します。

---

## 店舗 Windows PC 側でやること

- **一度だけ**：今回の変更を入れたビルド（手順2で作った `.exe`）を Windows PC に入れて再インストールしてください。
  （自動更新の土台となる設定ファイルがこのビルドから埋め込まれるため、最初の1回は手動インストールが必要です）
- 以降は**何もしなくてOK**。起動時に自動チェックし、新バージョンがあれば画面で知らせます。

---

## 補足・注意

- **署名について**：Windows では未署名でも自動更新は動きますが、初回インストール時に SmartScreen の警告が出ます（「詳細情報」→「実行」で進めます）。気になる場合はコード署名証明書の導入を検討してください。
- **開発モード（`npm run dev`）では自動更新は動きません**。インストール版でのみ有効です（安全のため）。
- **バージョンは必ず上げる**こと。同じ番号だと「最新です」と判定され更新されません。
- Mac 版（dmg）の自動更新には Apple の署名が必要です。今回は Windows 運用前提のため未対応です。
